# TransPilot — Material Design 3 Dynamic Color 主题改造计划

> **For Hermes:** 按 Task 顺序执行，每个 Task 完成后构建验证再继续。

**Goal:** 将 TransPilot 的 UI 主题从默认的 `lightColorScheme()` 升级为 Material Design 3 Dynamic Color（Material You），支持系统取色（壁纸颜色）并回退到蓝色主题。

**Current state:** 当前 Compose 主题使用 `lightColorScheme()` 无参调用，是 Material 3 默认色（靛蓝/灰白）。XML 主题 `Theme.LLMTranslator` 继承 `android:Theme.Material.Light.NoActionBar`，并未用于 Compose。

**Target behavior:**
- Android 12+ (API 31+)：从系统获取 Dynamic Color，支持 Light / Dark 自动跟随
- Android < 12：使用精心调配的蓝色调色板作为回退
- 状态栏/导航栏图标颜色适配（edge-to-edge 已开启，需调整状态栏文字颜色）

**Tech Stack:** Jetpack Compose, Material 3, `dynamicLightColorScheme`/`dynamicDarkColorScheme`, `isSystemInDarkTheme`

### Step 2: 创建 `Theme.kt`

```kotlin
package com.example.llmtranslator.ui.theme

import android.os.Build
import androidx.compose.foundation.isSystemInDarkTheme
import androidx.compose.material3.*
import androidx.compose.runtime.Composable
import androidx.compose.ui.platform.LocalContext

private val LightBlueColorScheme = lightColorScheme(
    primary = Blue40,
    onPrimary = Color.White,
    primaryContainer = Blue90,
    onPrimaryContainer = Blue10,
    secondary = BlueGrey40,
    onSecondary = Color.White,
    secondaryContainer = BlueGrey90,
    onSecondaryContainer = BlueGrey10,
    tertiary = Color(0xFF6F5A00),
    background = Color(0xFFFDFBFF),
    surface = Color(0xFFFDFBFF),
    onBackground = Color(0xFF1A1C2E),
    onSurface = Color(0xFF1A1C2E),
    surfaceVariant = Color(0xFFE0E2EC),
    outline = Color(0xFF74768B),
)

private val DarkBlueColorScheme = darkColorScheme(
    primary = Blue80,
    onPrimary = Blue20,
    primaryContainer = Blue30,
    onPrimaryContainer = Blue90,
    secondary = BlueGrey80,
    onSecondary = BlueGrey20,
    secondaryContainer = BlueGrey30,
    onSecondaryContainer = BlueGrey90,
    tertiary = Color(0xFFDBBD4C),
    background = Color(0xFF1A1C2E),
    surface = Color(0xFF1A1C2E),
    onBackground = Color(0xFFE3E2F9),
    onSurface = Color(0xFFE3E2F9),
    surfaceVariant = Color(0xFF46475B),
    outline = Color(0xFF9090A6),
)

@Composable
fun TransPilotTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    dynamicColor: Boolean = true, // 可开关，调试用
    content: @Composable () -> Unit
) {
    val colorScheme = when {
        // Android 12+: 使用系统动态色 (Material You)
        dynamicColor && Build.VERSION.SDK_INT >= Build.VERSION_CODES.S -> {
            val context = LocalContext.current
            if (darkTheme) dynamicDarkColorScheme(context)
            else dynamicLightColorScheme(context)
        }
        // 回退：蓝色调色板
        darkTheme -> DarkBlueColorScheme
        else -> LightBlueColorScheme
    }

    MaterialTheme(
        colorScheme = colorScheme,
        content = content
    )
}
```

> **注意:** `Color` 需要 `import androidx.compose.ui.graphics.Color` — 在 `Color.kt` 中已导入，`Theme.kt` 通过通配符 `import androidx.compose.material3.*` 不包含 `Color`，需要在 `Theme.kt` 顶部额外添加 `import androidx.compose.ui.graphics.Color`。

## Task 2: 更新 XML 主题（确保 system bar 正确适配）

**Objective:** 让 XML 的基础主题使用 Material 3 的 DayNight 主题，确保 Compose edge-to-edge 模式下 system bar 正确适配（避免 status bar 白色或不可见）。

**Files:**
- Modify: `app/src/main/res/values/themes.xml`

**Step 1:** 替换 `themes.xml` 内容

```xml
<?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.LLMTranslator" parent="Theme.Material3.DayNight.NoActionBar">
        <item name="android:statusBarColor">@android:color/transparent</item>
        <item name="android:navigationBarColor">@android:color/transparent</item>
    </style>
</resources>
```

> 注意：`Theme.Material3.DayNight.NoActionBar` 是 MDC 库中的主题，但我们当前没有显式依赖材料组件（Material Components for Android 的 XML 主题库）。如果编译报错找不到该主题，回退到：
> ```xml
> <style name="Theme.LLMTranslator" parent="android:Theme.Material.Light.NoActionBar">
>     <item name="android:statusBarColor">@android:color/transparent</item>
>     <item name="android:navigationBarColor">@android:color/transparent</item>
> </style>
> ```
> 因为 edge-to-edge 模式由 `enableEdgeToEdge()` 处理，XML 主题影响不大。

## 风险和开放问题

1. **`Theme.Material3.DayNight.NoActionBar`** — 需要 MDC Android 库依赖。如果不确定是否已存在，可以在构建时观察是否报错，报错则回退到 `android:Theme.Material.Light.NoActionBar` + 透明 system bar。
2. **Dynamic Color 在模拟器上的表现** — 模拟器通常没有壁纸，可能无法看到取色效果。真机测试更可靠。
3. **深色主题下的可读性** — 蓝色调色板的对比度需要实际验证，特别是 `surfaceVariant` 与 `onSurface` 的组合。
